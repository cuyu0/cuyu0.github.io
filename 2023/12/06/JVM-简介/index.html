JVM-简介

Created2023-12-06|Updated2023-12-06
|Post View:

概念

JVM（Java Virtual Machine）是Java虚拟机的缩写。它是一种能够执行Java字节码的虚拟机，是Java平台的关键组成部分之一。Java程序员使用Java编程语言编写应用程序，这些应用程序在JVM上运行而不是在特定硬件上直接运行。

JVM类型

  1. HotSpot JVM： HotSpot是由Sun Microsystems开发的Java虚拟机，它是Java平台的一部分。HotSpot以其高性能而著称，使用即时编译（JIT）技术将字节码转换为本地机器代码。
  2. JRockit JVM： JRockit是由BEA Systems（现在是Oracle Corporation的一部分）开发的Java虚拟机，专注于提供高性能和低延迟的Java应用程序执行。
  3. IBM J9 JVM： IBM的J9虚拟机是IBM公司的Java虚拟机实现，主要用于IBM的Java运行时环境，例如WebSphere。
  4. OpenJ9： OpenJ9是Eclipse Foundation的项目，也是一个开源的Java虚拟机实现，它具有高性能和低内存占用的特点。
  5. GraalVM： GraalVM是由Oracle Labs开发的一种新型的Java虚拟机，它支持多语言运行时，并且具有即时编译器（Graal编译器），可以提供优越的性能。

主要功能

JVM的主要功能包括：

  1. 字节码执行： Java源代码被编译成字节码，而不是机器特定的代码。JVM负责解释或即时编译这些字节码为机器码，以在底层硬件上执行。
  2. 内存管理： JVM负责管理Java程序的内存，包括堆内存（用于对象分配）和栈内存（用于方法调用和局部变量存储）。
  3. 垃圾回收： JVM提供自动垃圾回收功能，定期清理不再被引用的对象，释放内存。
  4. 即时编译（JIT）： 有些JVM实现采用即时编译技术，将字节码在运行时编译成本地机器码，以提高执行效率。
  5. 安全性： JVM提供安全性特性，包括字节码验证，以确保运行的代码不会违反Java语言规范。

JVM是Java跨平台特性的关键，使得Java程序能够在不同的操作系统上运行，只要有对应的JVM实现。这种跨平台性主要是因为Java程序不直接与底层硬件交互，而是通过JVM提供的抽象层进行交互。

JVM支持的是JVM字节码，任何能够编译成JVM字节码的语言，都可以在JVM上运行。

组成部分

JVM（Java Virtual Machine）的结构主要包括以下几个关键组成部分：

  1. 类加载器（Class Loader）：
    • Bootstrap Class Loader： 负责加载Java核心库，例如java.lang包。
    • Extension Class Loader： 负责加载Java扩展库，例如javax包。
    • Application Class Loader： 负责加载应用程序的类，是最常用的类加载器。
  2. 运行时数据区（Runtime Data Areas）：
    • 方法区（Method Area）： 存储类的结构信息、常量、静态变量等数据。
    • 堆（Heap）： 用于存储对象实例。所有线程共享堆，但是每个线程有自己的私有栈。
    • 栈（Stack）： 每个线程都有一个私有的栈，用于存储局部变量、操作数栈、方法出口等信息。
    • 程序计数器（Program Counter Register）： 记录当前线程执行的字节码指令地址。
    • 本地方法栈（Native Method Stack）： 用于支持Java以外的语言调用Java本地方法。
  3. 执行引擎（Execution Engine）：
    • 解释器（Interpreter）： 逐行解释执行字节码。
    • 即时编译器（Just-In-Time Compiler，JIT）： 将整个字节码编译为本地机器码，提高执行速度。
  4. 本地接口（Native Interface）：
    • 提供了一个接口，允许Java代码调用和被调用本地库中的方法。
  5. 本地方法库（Native Method Libraries）：
    • 包含了一组本地方法的库，这些方法可以通过本地接口调用。

JVM的位置

JVM是运行在操作系统上的，它与硬件没有直接交互

JVM字节码架构

JVM字节码是一种栈式架构。

Java 编译器（javac）将 Java 源代码翻译成字节码（bytecode），而字节码是一种与底层硬件架构无关的中间表示。字节码并不直接与特定硬件架构相关联，因此不涉及具体的计算机体系结构。

字节码采用的是一种栈式架构（stack-based architecture），它基于一种假设，即操作数和操作数结果都存储在一个操作数栈中。这种栈式架构是为了在不同的硬件平台上都能够保持一致性，实现 Java 跨平台执行的目标。字节码的执行是由 Java 虚拟机（JVM）来完成的，JVM 将字节码解释或编译成本地机器代码，以在特定硬件上执行。
